# PRD – نظام الترشيحات للمشاريع

## اسم الميزة
نظام إدارة وترشيح المستفيدين للمشاريع

---

# 1. الهدف من النظام

إتاحة إمكانية إنشاء مشاريع ترشيح جديدة من قبل الأدمن، بحيث يتمكن المندوبون من ترشيح اللاجئين ضمن قواطعهم وفق عدد محدد من المطلوبين، مع وجود صلاحيات مختلفة، تدقيق كامل (Auditing)، وتنبيهات عند تكرار ترشيح نفس العائلة أو أحد أفرادها.

---

# 2. الأطراف المستخدمة للنظام

| الدور | الصلاحيات |
|---|---|
| Admin | إنشاء المشاريع، تعديلها، مشاهدة جميع الترشيحات، الإضافة من كل القواطع، الاعتماد والإلغاء |
| مندوب | ترشيح الأشخاص فقط ضمن القاطع الخاص به |
| Supervisor (اختياري) | مراجعة واعتماد أولي |

---

# 3. إنشاء مشروع جديد (Admin)

## صفحة: إدارة المشاريع

يقوم الأدمن بإنشاء مشروع جديد يحتوي على:

### الحقول المطلوبة

| الحقل | النوع | الوصف |
|---|---|---|
| اسم المشروع | Text | اسم المشروع |
| تاريخ بداية الترشيح | DateTime | بداية استقبال الترشيحات |
| تاريخ نهاية الترشيح | DateTime | إغلاق الترشيحات |
| العدد المطلوب للترشيح | Integer | عدد الأشخاص المطلوب من كل مندوب |
| حالة المشروع | Active / Closed / Draft | حالة المشروع |
| ملاحظات | TextArea | ملاحظات إضافية |

---

# 4. صلاحيات المشروع

## المندوب

- يشاهد المشاريع المفتوحة فقط.
- يمكنه الترشيح ضمن القاطع الخاص به فقط.
- لا يمكنه رؤية ترشيحات القواطع الأخرى.
- يمكنه تعديل ترشيحاته قبل الاعتماد النهائي.

## الأدمن

- مشاهدة جميع الترشيحات.
- الإضافة من أي قاطع.
- تعديل وحذف واعتماد أي ترشيح.
- تجاوز بعض القيود مع ظهور Audit Log.

---

# 5. صفحة الترشيحات للمندوب

## شكل الصفحة

جدول يحتوي على عدد الصفوف بحسب العدد المطلوب.

مثال:
إذا العدد المطلوب = 10
يظهر 10 صفوف للترشيح.

---

## أعمدة الجدول

| العمود | الوصف |

| رقم الهوية | يتم تعبئته تلقائياً |
| رقم الجوال | يتم تعبئته تلقائياً |
| اسم العائلة | تلقائي |
| القاطع | تلقائي |
| الحالة | تنبيه / معتمد |
| ملاحظات | Optional |

# 7. منع التكرار والتنبيهات

## السيناريو

إذا قام المندوب باختيار:

- نفس الشخص مسبقاً.
- أو أي فرد من نفس العائلة.

يظهر:

### Warning Message

> "يوجد ترشيح سابق لنفس العائلة أو أحد أفرادها، هل تريد الاستمرار؟"

### السلوك

- التحذير لا يمنع الحفظ.
- يمكن للمندوب المتابعة.
- يتم تسجيل ذلك في الـ Audit Log.

---

# 8. قواعد العمل (Business Rules)

## قواعد الترشيح

### يمنع:

- تجاوز العدد المطلوب للمندوب.
- الترشيح بعد انتهاء تاريخ المشروع.
- حذف ترشيح معتمد إلا بصلاحية Admin.

### يسمح:

- الأدمن يمكنه تجاوز القاطع.
- الأدمن يمكنه الإضافة حتى لو يوجد تكرار.
- الأدمن يمكنه الاعتماد والإلغاء.

# 17. ملاحظات مهمة للتنفيذ

- يجب أن تكون جميع العمليات Transactional.
- يجب منع Race Conditions أثناء الترشيح.
- يفضل استخدام Soft Delete.
- يفضل استخدام RowVersion للـ Concurrency.
- جميع العمليات الحساسة تسجل في Audit Log.
- البحث داخل Select2 يجب أن يكون سريعاً حتى مع بيانات كبيرة.
